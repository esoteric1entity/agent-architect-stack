The Premise

A clear statement of purpose, the problems we plan to solve, and what makes this stack different from anything else in the agent-infrastructure space.

Purpose

To give AI agents a stable, inspectable, installable infrastructure layer — the same way Kubernetes gave services a stable substrate. The Agent Architect Stack is a set of peer-sibling modules an agent harness can install to gain persistent memory, a defensive runtime, and (over time) more. Each module is independent; the umbrella is just a way to discover them together.

Problems we solve

  • Agents forget between sessions. The Memory branch gives your agent a persistent vault + knowledge graph so context survives restarts, machine moves, and context-window overflows.
  • Skills and tools are an unvetted supply chain. The Security branch gives your agent a 3-tier automated + 5-layer manual vetting pipeline, runtime bash + write guards, and an append-only audit log — so installing a new skill doesn't compromise the runtime.
  • No standard for cross-agent context. The Memory + Security branches together produce a portable context format (the vault + audit log) that any agent harness can read — a first step toward cross-agent federation.
  • Agent projects are monolithic or scattered. The umbrella model is neither. Each branch is a complete product; the umbrella is a discovery surface, not a runtime dependency.

What makes this stack unique

  • Pick-and-choose installation. Install one branch, install some, install all. No forced dependencies, no monolithic install. Each branch is independently versioned.
  • Beyond built-in context. Harness memory is session-bound and opaque. The Memory branch gives you a persistent, human-readable vault + knowledge graph that survives restarts, moves across machines, and is structured for cross-agent reuse — not just the last N turns of chat.
  • Data lives in files you own. The Memory branch stores everything as plain Markdown + JSON in a vault you can open in Obsidian, grep, version-control, and back up with cp -r. No proprietary database, no vendor lock-in.
  • Defense by design, not just by theory. The Security branch's architecture spans 8 layers; the current release ships six of them — Layers 1, 2, 3, 4, 6, and 7 (skill/tool vetting, input sanitization, structured-output enforcement, runtime hooks, audit logging, and configuration), covered by an extensive test suite. Layers 0 (cron rotation) and 5 (network egress) are designed and roadmapped.
  • Schema-enforced memory hygiene. Decisions, learnings, and daily logs all follow documented schemas with lint tools that surface problems for human review (never auto-mutate — surface-only by design).
  • Architected by hand. Built with agents, for agents. The v0 release is the collective work of multiple agent sessions across multiple machines under human direction — a meta-demonstration of the cross-agent collaboration the stack itself enables.

What it does

The Agent Architect Stack turns a stateless LLM into a stateful, inspectable, defensible agent by giving it a persistent memory vault, a vetting pipeline for new tools, runtime guards against untrusted input, and an audit log you can review.

Memory Released

Persistent memory + knowledge graphs + Obsidian vault + prompt compression.

  • Survives context resets (your agent remembers between sessions)
  • Searchable as a graph (entities, relationships, temporal facts)
  • Human-readable vault (Obsidian + Templater)
  • Prompt compression at a quality-preserving threshold
  • Auto-detects Claude Code, OpenClaw, or generic 9-root-file harness

Security Released

8-layer defensive overlay for AI agents: skill vetting, input sanitization, runtime hooks, network egress, audit.

  • 3-tier automated + 5-layer manual escalation for new skills
  • 4-layer input sanitization (structural, content, encoding, context)
  • Bash + write guards (Python + bash, cross-platform-equivalence tested)
  • Append-only audit log with SHA-256 chain integrity
  • Anti-prompt-injection via structured output enforcement
Repo: esoteric1entity/agent-shield — public
License: Apache-2.0
Status: v0.2.1 alpha — Layers 1, 2, 3, 4, 6 & 7 shipped (vetting, sanitize, structured prompts, runtime hooks, audit, config); Layers 0 & 5 roadmapped

You Choose

The modular consumer principle: you get exactly what you need, with zero lock-in. The branches share conventions, not dependencies.

1

Independent installation

Install Memory alone. Install Security alone. Install both. No auto-installs, no forced dependencies. Each branch is a complete product on its own.

2

Independent versioning

Memory and Security ship on their own timelines. You can install either branch today. Compatible versions are documented per release.

3

Better together

When you install both, Memory's audit logs feed Security's anomaly detection. When you install just one, that branch is the whole product. The umbrella grows; you don't have to.

Quick Start

Two branches, two install paths. Pick the branch you want; each is independent.

Security branch

One-line install

pip install ai-agent-shield

# Verify the install
python -c "import agent_shield; print(agent_shield.__version__)"

# Then wire the guards into your harness (consent-gated):
# https://github.com/esoteric1entity/agent-shield/blob/main/INSTALL_AGENT.md

Memory branch

Four ways in — Install the Ultimate Memory Stack however you prefer:

Door 1 — Script (Linux / macOS / WSL · Windows)
git clone https://github.com/esoteric1entity/ultimate-memory-stack.git

cd /path/to/your/workspace            # where the memory stack should live
/path/to/ultimate-memory-stack/setup-memory-stack.sh    # or: --target <dir>
/path/to/ultimate-memory-stack/verify.sh

# Flags: --minimal · --addon <name> · --no-templater · --compliance=<preset> · --yes
# Windows: setup-memory-stack.ps1 (same options, PowerShell-style: -Minimal, -Addon, -Compliance …; needs Python 3.8+)
Door 2 — Tell your agent (any harness)
git clone https://github.com/esoteric1entity/ultimate-memory-stack.git

# Then tell your agent — Claude Code, OpenClaw, or any capable harness:
#   "Install this — read INSTALL_AGENT.md and follow it."
Door 3 — Claude Code marketplace

⚠️ Run these inside Claude Code, not your shell — Claude Code must be installed & authenticated.

# 1. In Claude Code — add the marketplace + install the plugin
/plugin marketplace add esoteric1entity/ultimate-memory-stack
/plugin install ultimate-memory-stack@ultimate-memory-stack

# 2. Exit Claude Code (/exit or Ctrl-D)
# 3. In your shell: cd to the project where the memory should live
# 4. Relaunch Claude Code from inside that directory, then run:
/install-ultimate-memory-stack

Note: Existing memory/ store in that project? Back it up first — or use the script/agent door, which preserves it automatically.

Door 4 — no tooling at all

Drag the common-specs/ and general-edition/ folders into your workspace, then paste The Prompt into your agent — it runs the setup wizard. Full manual steps: the Install Guide.

Whatever the door, the installer refuses to mix your memory into the package tree and records exactly what it did in a manifest.

Harnesses, models, and hardware in the agent ecosystem worth knowing about. Some of this we run ourselves; the rest we've checked out and found legitimate — real, functioning, and worth your time. We don't list vaporware, and nobody pays to be on this page.

Harnesses

(5)

The agent runtimes that drive tool calls, memory, and channels — this stack is built to work with any of them.

Claude Code

The honest admission: this entire stack is built and shipped inside it, every day. Big-company product, undeniably the tool in our own hands.

OpenClaw

The harness we build our own adapters against — both Memory and Security branches ship official OpenClaw integrations.

Hermes Agent

Nous Research's memory-first alternative — agents that build and refine their own skills over time. MIT-licensed.

OpenCode

A terminal-native coding agent — MIT-licensed, LSP-aware, works with any model provider you point it at.

NemoClaw

NVIDIA's hardened wrapper around OpenClaw — sandboxes the agent inside OpenShell, supports Hermes as an alternate agent too.

Models & inference

(7)

Where the actual reasoning comes from, and how it gets served.

Ollama

The local model runtime we actually use in our own benchmark work.

llama.cpp

The C/C++ engine underneath most local-inference tools, Ollama included — this is literally what our own benchmark study's quantization work runs on.

vLLM

The production-grade serving engine — PagedAttention and continuous batching for squeezing real throughput out of a GPU fleet.

OpenRouter

One API, 400+ models — useful when you don't want to hardcode a single provider.

Hugging Face

The model and dataset hub — where most open weights actually live.

Groq

LPU-based cloud inference — purpose-built silicon for fast, cheap token generation, not a repackaged GPU.

NVIDIA Nemotron

NVIDIA's open model family, open weights and training recipes, tuned for agentic workloads and local deployment.

Marketplaces & Skill Hubs

(2)

Where you get skills and plugins once the harness is running.

ClawHub

The community skills distribution hub for OpenClaw — where we're planning to publish this stack's own skills next.

Claude Code Plugin Marketplace

Anthropic's native plugin/marketplace mechanism — it's literally how the Memory branch installs today (/plugin marketplace add).

Developer Platforms

(2)

Where you actually go to build — API access, docs, and dev programs.

Claude Developer Platform

Anthropic's own API and developer platform — the one this entire stack is built to run on top of.

AMD Developer

Free AI developer program — ROCm tooling, cloud credits, and training for building on AMD hardware.

Context & Retrieval

(4)

Getting the right context to the model, not just more of it — whether that's a tool call or a retrieved chunk.

Model Context Protocol (MCP)

The open standard for connecting agents to tools and data — this stack is hook-based rather than MCP-based by design, but it's the protocol most of the ecosystem is converging on.

LlamaIndex

The leading RAG-specific framework — indexing, retrieval, and query pipelines built for exactly this job.

Contextual Retrieval

Anthropic's technique for fixing RAG's biggest failure mode — chunks that lose their surrounding context. Directly relevant to how the Memory branch thinks about retrieval.

LangChain

The broadest framework in the space — not RAG-only, but its retrieval tooling is where most people first encounter the pattern.

Biotech & Life Sciences

(2)

Where agent infrastructure meets the lab bench.

LabClaw

An operating layer for LabOS, built for the Stanford–Princeton line of AI co-scientist work — agents doing real lab research, not just suggesting it.

BIO26 Healthcare Special Address

NVIDIA's Kimberly Powell at BIO International Convention 2026, on the BioNeMo Agent Toolkit and the shift from AI that suggests science to AI that runs it.

Benchmarks

(1)

How you tell a capable agent from a confident one.

SWE-bench

The standard for measuring whether an agent can actually fix real GitHub issues, not just talk convincingly about code.

Hardware for local AI

(3)

For running real models on your own machine, not a rented one.

NVIDIA DGX Spark

A desktop-sized AI supercomputer — for running real local models, not toy demos.

AMD Ryzen AI Max ("Strix Halo")

128GB unified memory in a desktop APU — a serious x86 option for local inference.

Mac Studio

Apple Silicon with genuinely high unified memory — built for workloads like this, not just video editing. (The Mac Mini is the wrong pick here: not enough memory bandwidth for real local models.)

Learning

(4)

Free courses that actually issue a certificate — not just a "you watched the video" badge. Paid options may get their own list later.

Anthropic Academy

Free courses on Claude, the API, Model Context Protocol, and Claude Code itself — straight from the source, certificates included.

NVIDIA Deep Learning Institute — Free Courses

Self-paced AI and accelerated-computing courses; select ones issue an NVIDIA DLI certificate on completion.

Google Cloud — Introduction to Generative AI

A ~45-minute no-cost primer; stack it with two more free courses for the full Generative AI Fundamentals skill badge.

Kaggle Learn

Genuinely 100% free, no paywall trick — real Python notebooks, real model calls, certificates on completion.
